## Side finding: `send.py:155` keystroke is unrelated to the banned reply-body clipboard path

The task asked me to check what flows through `compose/send.py`'s `keystroke "v" using command down`. It belongs to `_send_html_email`, the `compose_email(body_html=...)` HTML-formatting path, added in a dedicated commit (`34aa7ae`/`f1c57a5`, "feat: add HTML email support via NSPasteboard clipboard injection"). It deliberately places HTML on `NSPasteboard`, tabs into a **new, empty** compose window's body, pastes, and then restores the prior clipboard contents (`send.py:113-114`, `165-169`). This is a different tool (`compose_email`, standalone new mail) for a different purpose (rich HTML formatting) than the reply body path, was never reverted, and is not the clipboard mechanism the AGENTIC-1214/781 history note bans (that ban is specific to typing a **reply** body into an **existing native reply window**, where a clipboard swap risked leaking into the wrong thread). No action needed on this branch; noted so the implementation phase does not mistake it for leftover dead code from the reverted reply-clipboard attempt.
