# AGENTIC-1277 and AGENTIC-1191 closeout, 2026-07-11

**Branch:** `fix/agentic-1277-compose-draft-recipient-verification`
**Release:** v3.11.3
**State:** committed locally with rebuilt release artifacts, ready to push

## Goal

Make the compose draft smoke prove the exact persisted recipient payload and
cleanup identity, while correcting the reproducible bounded Inbox performance
regression without turning capped Drafts visibility into a false negative.

## Shipped changes

- `draft-verify-smoke` now requires the exact normalized To-recipient set,
  returns redacted mismatch evidence, and retains every unverified candidate.
- Internal cleanup validates the generated subject, exact recipient set, and
  body sentinel in the same Drafts AppleScript transaction before deletion.
  Drift, ambiguity, or mismatch fails closed and retains the draft.
- Reply-state snapshots cap Drafts scans at 50 and emit total/truncation
  metadata. A match in the scanned window remains `true`; a capped nonmatch
  is `null`, never a false `has_draft=false` claim.
- The CLI performance battery now reads the structured mailbox envelope's
  returned count when scaling its metadata threshold.
- Archived the shipped AGENTIC-1214 and reply-state-annotation workstreams,
  and refreshed task pointers for the current repository state.

## Verification

- Focused CLI, compose, reply-state, reply-state-wiring, and search tests pass.
- `bash tools/gates/dev-check.sh release` passed: ruff, strict mypy, all
  1,551 tests, expected-count gate, manifests, rebuilt plugin zip and MCPB,
  artifact parity, MCPB validation, and wrapper surface checks.
- Live read-only quick perf check passed on Cayman Agentic Assets: metadata
  3448.6 ms against 4800 ms, Inbox 4879.9 ms against 5000 ms, with a 50-Draft
  scan marked truncated.
- Live draft-only smoke to `cayman@agenticassets.ai` verified the exact
  recipient set and identity-guarded cleanup. No email was sent.

## Decisions

- Kept the public `manage_drafts` API unchanged. The smoke uses an internal
  identity-guarded helper because a numeric Exchange Drafts ID alone is not a
  durable deletion handle.
- Kept the 50-Draft performance cap, but exposed incomplete knowledge as
  `null` instead of suppressing a potentially existing draft.

- AGENTIC-1192 Archive exact-ID reply lookup, AGENTIC-995 deterministic
