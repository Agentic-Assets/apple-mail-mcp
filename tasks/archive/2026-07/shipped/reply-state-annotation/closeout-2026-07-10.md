## What shipped

- **`was_replied_to` (bool, always present)** on every discovery row of the 8 primary
  read/triage tools (`list_inbox_emails`, `search_emails`, `get_email_by_id`,
  `get_email_by_ids`, `get_email_thread`, `get_needs_response`, `inbox_dashboard`,
  `get_inbox_overview`), read from Mail's native `was replied to` flag inside the
  existing per-message property pass. No parameter gates it.
- **`has_draft` (true/false/null)** on the same rows via one bounded Drafts snapshot
  per account per call (`DRAFT_LOOKUP=75`, header correlation capped at
  `DRAFT_SNAPSHOT_HEADER_CAP=10`), correlated by In-Reply-To/References match or
  normalized subject + recipient-equals-sender + date ordering. `include_draft_state:
  bool = True` escape hatch; `null` means scan skipped or errored, never silently
  false. Top-level `draft_scan` status object in JSON; `[REPLIED]` / `[HAS DRAFT]`
  markers in text modes.
- **`get_needs_response` excludes already-handled mail by default** with visible
  `skipped_replied_count` / `skipped_drafted_count` and `include_already_replied` /
  `include_drafted` overrides; fail-open on draft-scan error. Legacy
  `check_already_replied` Sent scan kept as opt-in extra layer.
- **`exclude_drafted` (default False)** added to `list_inbox_emails` /
  `search_emails`; `exclude_replied` reworked to the native flag; `flag_replied`
  deprecated but accepted.
- **New modules:** `core/reply_state.py` (snapshot builder, correlation rule,
  localized Drafts-mailbox resolver, `resolve_has_draft`, `reply_state_tags`),
  `tools/reply_state_wiring.py` (shared annotator, `date_field`-parameterized),
  `tools/smart_inbox/reply_state_glue.py`. **Deleted dead code:**
  `tools/inbox/replied.py` (native flag replaced the Sent-scan path) and the
  transitional `tools/search/reply_state_glue.py`.
- **Skills/docs:** canonical `pre-draft-verification.md` and
  `recent-first-triage.md` rewritten around the row-level check as the primary
  pre-draft duplicate guard (thread check demoted to fallback), synced into all four
  packaged skills; `plugin/skills/CLAUDE.md`, three SKILL.md files, templates,
  examples, `tools/CLAUDE.md`, `docs/CLAUDE-conventions.md`,
  `docs/AGENT_LIVE_TESTING.md`, mcpb `get_needs_response` description updated.
  Skill-reviewer pass ran; all 10 findings (1 blocker, 6 major, 3 minor) fixed and
  re-synced.

## Anomalies observed live (not regressions; queued in forward queue)

1. `quick-check` static thresholds (metadata 2s, inbox 5s) miss on this
   100-mailbox account (3.4s / 6.9s measured). The metadata miss predates this
   branch; the inbox case now includes the ~2s draft snapshot and needs a
   threshold rescale or an `include_draft_state=False` baseline case.
2. Mail tracks the native answered flag per mailbox copy: the same `message_id`
   read via `get_email_thread` (an "All Mail" copy) reported
   `was_replied_to=false` while the INBOX copy reported `true`. Each tool
   reports its own copy correctly; cross-tool consistency note needed in docs.
3. `draft_scan.accounts` is a dict list on the annotated multi-account tools
   but a plain string list on single-account `get_needs_response` (pinned by
   test, documented in the module docstring). Align in a future minor.
